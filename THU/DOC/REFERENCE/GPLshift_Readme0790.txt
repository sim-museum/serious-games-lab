INSTALLING: (what you need to do)

	
1st: If you are using a old version of GPLshift then move these files from 
      ...\GPL root to another dir: (e.g. to \GPLshiftOLD  or  \GPLshiftV0.790) 
      GPLshift.dll, GPLshift.ini!, GPLpatch.exe/GPLshift-Patch.exe, 
      & GPLshift_ReadMe0790.txt (GPL-BACK.EXE)
       
2nd: Then take a copy of GPL.EXE, (also if you don't have a previous version of 
      GPLshift) to same/another dir, to be sure.. and so you can come back to
      the old version, if the new v0.7.9.0 does not work.  

NB:  If you have already patched with GPLpatch.exe v0.5 (or higher), then you
      don't need to do it again, but remember to replace ALL file with them 
      found in the ZIP, also the GPLshift.INI! because it can have been change

3rd: Unpack GPLshift0790.ZIP, move GPLpatch.exe, GPLshift.DLL, GPLshift.ini, 
     GPLshift_ReadMe0790.txt GPLshift_WhatsNew0790.txt to GPL dir. where you have
     GPL.EXE (e.g. C:\SIERRA\GPL\) I suggest you unzipper direct in GPL root!

4th: If you need to run others patch (e.g. a NO_CD_Patch) please do it BEFORE 
      running GPLpatch.exe, to avoid a message like  "CRC32 check error"    
      from these programs

5th: Start GPLshift_Patch.exe, if you haven't patched before with v0.5 (or higher)

             First:  The patch will check/copy the GPL.EXE to e.g:
		      GPL.EXE-1201us-BeforeGPLshiftPatch0790.BAK
                                          or
		      GPL.EXE-1202eu-BeforeGPLshiftPatch0790.BAK
		                          or
		      GPL.EXE-2004Demo-BeforeGPLshiftPatch0790.BAK

             Second: Making some necessery chance in GPL.EXE
                      for running with this utility's Shifters/TrackIR support
		      and the other functions	

          ***  You will only need to run this patch v0.790 once  ***
                   
                     If you want to unpatch GPL.EXE later, then just start 
		      GPLshift_Patch.exe again


6th. Change the setup in GPLshift.ini if you find it necessary

7th. And your GPL should be ready now, for running with ACT LABS or FrexGP 
     Shifter, etc.

Special Notice before starting the racing:

GEM+ Tool:
 If you have installed GEM+ BEFORE the GPLshift Patching, then you
 have to delete all GPLc*.exe files (e.g GPLc66.exe, GPLc67.exe etc.) 
 BUT DO NOT DELETE GPL.EXE! - next time you run GEM+ it should recreate the
 deleted GPLc*.exe files base on the patched GPL.EXE 

---------
If you don't use a analogic game device for clutch then remember in the
  GPLshift.ini to keep/set: 

[ Clutch Device ] 
Clutch_Threshold = 0      ;(OFF)

 otherwise any gear shift will be bloked  (button for clutch will not help)

----------
If you are using shortcut to GPL.EXE, then remember this in Windows
 shortcut properties/Shortcut: 

 Target:     x:\GPLrootDIR\GPL.EXE 
 Start in:   x:\GPLrootDIR    <-- So GPL.EXE can find GPLshift.dll under upstart 

----------
Don't set the number of sounds too low in GPL Options/Sound & Gameplay
 because it can stop other sounds when running a big ShftErr.wav file 
 
----------
If you are using GPLshift's Shifter unit support, then remember to 
 disable the AUTOMATIC SHIFTING in GPL OPTIONS\SOUND & GAMEPLAY 
 to avoid strange shifting behavior
 
And exit ACT LABS / T. Grandgent Shifter utility ALSHIFT, to be sure it don't 
 interfere with GPLshift+ Gear shifting
